⚙️ 一、基础系统设置
控制程序的运行模式、帧率、窗口等全局行为。
参数名称范围 / 类型默认值说明
推理帧率上限 (AimFpsLimit)0 ~ 9990 (全速) V3.7 推理线程帧率上限。0 = 全速，节拍随截图管线（不超截图上限）；>0 时按设定值限频。引擎内部硬上限 1000 FPS。默认 0 即可
异步模式 (EnableAsync)开关开启 截图与推理异步执行。异步模式下推理帧率可超过截图帧率（复用最新帧），由 GPU 算力决定。强烈推荐开启
推理窗口 (ShowInferenceWindow)开关关闭显示实时推理画面（调试用），会增加额外 GPU 占用。
调试模式 (DebugMode)开关关闭输出详细性能日志（FPS / Loop / Infer / Cap 耗时），仅开发调试使用。
反截图 (EnableAntiScreenshot)开关开启防止推理窗口与控制台被游戏内截图/录屏软件捕获。
一键安全模式按钮—V3.7 网页面板按钮：清日志 + 关闭调试模式 + 隐藏日志窗口。
V3.7 帧率架构说明 网页面板状态栏中的 FPS 是推理帧率（AimbotThread 每秒实际处理的帧数）。
采集卡模式下额外显示 CCap（采集卡实际帧率）；WATESender 模式下显示 NRcv（网络接收实际帧率）。
推理 FPS 默认全速：通常采集 240Hz + 全速推理 → ~500–900fps（取决于显卡）。更高推理频率 = 更强锁定 + 更平滑移动。
🖥️ 二、截图模式与采集配置
选择如何获取游戏画面。不同模式适用于不同场景。
参数名称范围 / 类型默认值说明
截图模式 (CaptureMode)1 / 2 / 4 / 52 (DXGI) V3.7 已精简
1 = GDI (CPU)：兼容性好但受显示器刷新率限制，每帧独立私有 DC
2 = DXGI (GPU)：高性能，推荐首选；HDR / scRGB 自动 sRGB 校正
4 = 采集卡：USB 采集卡（DSHOW，YUY2 / NV12 / MJPEG 直通）
5 = WATESender：QUIC（msquic）+ WATE/1 二进制帧；统一替代旧 5/6/7 三种网络模式

采集卡配置（模式 4）

参数名称范围默认值说明
设备索引0, 1, 2...0采集卡设备编号，可通过「扫描设备」获取。
采集宽度640 ~ 38401920采集分辨率宽度。
采集高度480 ~ 21601080采集分辨率高度。
编码格式YUY2 / NV12 / MJPEGMJPEGV3.7 YUY2 / NV12 为无压缩直通；MJPEG 为大多数 USB 采集卡协商 120/144/240fps 必选。
采集帧率30 ~ 24060采集卡输出帧率。截图线程以此为上限，与 AimFpsLimit 解耦。高刷采集卡建议 144/240。
裁切尺寸0 ~ 192000 = 使用 AI_InputSize 自动裁切中心区域。

V3.7 采集卡高帧率注意事项 1. 确保采集卡硬件支持目标帧率（HDMI 2.0 最高 4K@60 / 1080p@240）
2. MJPEG 编码协商帧率，大多数 USB 采集卡在 MJPEG 下可支持 120/144/240fps

4. 推理 FPS 默认全速会高于采集帧率，多出的周期复用最新帧（异步模式）

🧠 三、AI 引擎参数

类别总数 (AI_TotalClasses)1 ~ 648模型输出的目标类别数量。V3.7 YOLOv5/v8 形状自动消歧，加载模型后自动识别。
推理后端 (AI_UseCUDA)0 / 10 (DirectML)V3.7 网页面板下拉框：
0 = DirectML（N卡 / A卡 / Intel 通用）
1 = TensorRT for RTX 1.4仅 N 卡 RTX 系列

🎯 五、瞄准参数（全局）
全局生效的瞄准相关参数，影响所有配置组和按键绑定。
参数名称范围默认值说明
全局移动频率 (MoveHz)0 ~ 1000 HzV3.7 200全局鼠标移动发送频率。0 = 使用协议内置频率；>0 = 覆盖协议特定频率。推荐 200~400
瞄准附加延迟 (AimExtraDelayMs)0 ~ 10 ms0每次移动指令后额外等待的毫秒数。0 = 无延迟。
单次最大移动 (MaxMovePerStep)1 ~ 200 px50每次发送的最大移动像素数。防止单次移动过大导致抖动。关键参数
持续锁定 (Aim_Continuous)开关关闭开启后，松开按键也继续瞄准（直到目标丢失）。
V3.7 抖动抑制说明 旧版 B+ 专属的「反抖动」「额外平滑」两个开关已下线，其能力被合并到 PID 输入侧 Smooth、Smooth_Deadzone、近距 closeBoost smoothstep 过渡和 PID 输出 EMA 中，无需手动开启。
若仍出现抖动，请按 PID 节末尾「调参口诀」依次提高 D / 提高 Smooth / 降低 P 即可。
📐 六、PID 瞄准算法（按键独立）
PID 控制器是瞄准系统的核心算法。每个按键绑定有独立的 PID 参数。合理调节可实现丝滑拉枪 + 死区硬锁。

X 轴参数

参数名称范围默认值说明
P-X (比例增益)0.00 ~ 1.000.120比例系数。越大拉枪越猛，过大会震荡/超调。核心参数
I-X (积分增益)0.00 ~ 2.000.300消除稳态误差，使准心精确到达目标。过大导致积分饱和。
D-X (输出阻尼)0.00 ~ 0.100.020值越大，输出变化越被抑制 → 抗振荡。0=无阻尼，0.02=中等，0.05+=强阻尼。抖动时增大
限幅 X (LimitX)0.00 ~ 1.000.620X 轴积分限幅，防止积分项过大。

Y 轴参数

参数名称范围默认值说明
P-Y (比例增益)0.00 ~ 1.000.100通常略小于 P-X，垂直方向需要更稳。
I-Y (积分增益)0.00 ~ 2.000.300Y 轴积分系数。
D-Y (输出阻尼)0.00 ~ 0.100.020Y 轴输出阻尼系数。
限幅 Y (LimitY)0.00 ~ 1.000.520Y 轴积分限幅。

平滑与磁吸

参数名称范围默认值说明
平滑 (Smooth)0.10 ~ 1.000.100.10 = 无平滑（最硬，响应最快）；1.00 = 最柔（丝滑但延迟高）。支持两位小数微调。核心参数
移动死区 (Smooth_Deadzone)0.0 ~ 50.0 px1.5误差 ≤ 该值时准心完全静止，禁用曲线/平滑/移动输出。防瞄到人后画圈
磁吸增益 (AimMagnet)0.0 ~ 10.02.5死区内的锁定力度。1.0=无增益；越大准心锁定越"死"。推荐 2.0~4.0

速度前馈（FollowGain）V3.7 升级

v2.0 引入 / v3.7 调优：旧版作为远距 P 增益（[1,3]），新版语义为目标速度前馈 [0,1]。
参数名称范围默认值说明
速度前馈 (FollowGain)0.00 ~ 1.000.00公式：desired += FollowGain × 目标帧速度。0 = 关闭；1 = 理论零滞后跟枪。配置文件字段名沿用旧 FollowGain。想跟快速横移敌人时启用 0.3~0.6
迁移提示 旧版 V2.9.5 配置中 FollowGain 数值若 ∈ [1, 3]，加载时会被自动 clamp 到 [0, 1]；建议手工重新调整该值。
PID 调参口诀 拉枪太慢？ → 提高 P 值 & 降低 Smooth
拉枪震荡/超调？ → 降低 P 值 & 提高 D 值
跟枪不紧？ → 提高 AimMagnet & 提高 I 值
跟横移目标滞后？ → 提高 FollowGain（0.3~0.6）
准心抖动？ → 提高 Smooth & 提高 D 值
默认参数已经过反复优化，建议新手先使用默认值体验。
👁️ 七、FOV / 检测参数（按键独立）
控制 AI 检测的敏感度和瞄准区域。每个按键绑定可独立设置。

瞄准高度 高 (Aim_HeightMax)0.00 ~ 1.000.20与 HeightMin 相同时固定位置；不同时为随机范围。

HeightMin/Max 示例 0.20 / 0.20 → 固定瞄头部 20% 位置
0.15 / 0.30 → 在头部 15%~30% 范围内随机瞄准（更自然）

🛡️ 十二、屏蔽参数

多游戏 / 多枪压枪曲线管理 V3.7

RCS 为「游戏 → 枪械 → 阶段 (Stage)」三级树形结构。每把枪可设置多个阶段：

字段说明
游戏 (RCS_Games)独立配置游戏，下拉框切换；每个游戏内独立维护枪械列表。
枪械 (Guns)每把枪自定义名称与阶段曲线，在游戏面板中下拉切换。
Count（子弹数）该阶段持续的子弹数量。
X（水平补偿）正值向右补偿，负值向左。
Y（垂直补偿）正值向下补偿（抵消上跳）。默认 1.0。
🏷️ 十五、类别筛选与排序
控制检测哪些类别的目标、优先级排序、以及部位分离。按键绑定级别可独立设置。
参数名称类型说明
类别勾选 (ClassMask)复选框按模型类别 ID（0~63）勾选要检测的目标类型。
类别排序 (Str_ClassSort)字符串格式：0-1-2-3-4，前面的优先。
部位分离 (Str_PartsSep)字符串格式：0.2,0.3-0.2（头/身体分离高度）。

转火类别 (Str_SwitchClass)字符串限定转火只在特定类别间发生。

❓ 十七、常见问题与调参建议
Q：瞄准太慢、拉枪跟不上？ A：提高 P-X / P-Y → 降低 Smooth → 适当提高 AimMagnet。
Q：瞄准时准心抖动（左右晃）？ A：提高 D 值 → 提高 Smooth → 降低 P 值。
V3.7 旧版「B+ 反抖动 / 额外平滑」开关已下线，等效逻辑被并入 PID 输入侧 Smooth 与近距 closeBoost smoothstep 过渡，无需手动启用。
Q：瞄准超调（准心冲过目标再拉回）？ A：降低 P 值 → 提高 D 值 → 降低 I 值。
Q：跟横移敌人滞后？ A：V3.7 启用「速度前馈 (FollowGain)」0.3~0.6，等效让 PID 在跟随同时叠加目标速度补偿；切勿超过 1.0。
Q：瞄准有延迟感？ A：确保 AimExtraDelayMs = 0 → AimFpsLimit 设为 0（全速）→ 截图模式选 DXGI → 异步模式开启。V3.7 默认值即最优。
Q：采集卡模式帧率只有 60fps？ A：编码格式选 MJPEG（V3.7 默认），并把「采集帧率」设为采集卡支持的最大值（如 144/240）。状态栏 CCap 显示实际采集帧率。
Q：旧版 5/6/7 网络模式（加密 UDP / 普通 UDP / TCP）去哪了？ A：V3.7 已合并下线 三种模式统一替换为新模式 5「WATESender」：QUIC（msquic）+ WATE/1 二进制帧（JPEG / H.264）。首次接入会自动从 WATESender 的 /api/ca.pem 拉取证书并写入用户 Root，后续连接自动信任。
Q：旧版的「预判移动」没了？ A：V3.7 整体下线。其能力由 PID + 速度前馈 (FollowGain) 与跨帧曲线轨迹学习共同接管，配置更直观且不会过度补偿。
Q：开启 WATE 后游戏掉帧/卡顿？ A：V3.7 默认配置已做全面性能平衡（OpenCV/ONNX 线程限制、GPU Keepalive 条件脉冲、TRT 1 帧延迟流水线）。如仍掉帧：
1. AimFpsLimit 设为 0=全速但留出 GPU 空隙；如显卡低端可手设 150~300
2. 截图选 DXGI；HDR 显示器自动 sRGB 校正无需手动
3. 关闭「推理窗口 (ShowInferenceWindow)」减少额外 GPU 占用
Q：MAKCU / B+ 串口连接不上？ A：用「扫描设备」找正确 COM 口 → 默认波特率 115200（CH343 设备最高 4 Mbps，按设备规格升档）→ 重新点击「连接设备」。
Q：扳机不自动射击？ A：确保「扳机开关」已开启 → 需按住瞄准键时扳机才生效 → 检查触发范围是否合理。V3.7 若想纯扳机不带吸附，关闭「磁性扳机」即可。
新手推荐配置 初次使用建议保持所有参数默认值。默认 PID 已经过大量调试。
先熟悉基本操作：选模型 → 连设备 → 启动运行 → 按住右键瞄准。
